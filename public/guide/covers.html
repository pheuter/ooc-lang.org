 Covers got to be my favorite aspect of ooc. In essence, a cover is nothing more than
a C typedef on rails. Covering primitive types extends functionality and allows for higher-level
manipulation. Every type in ooc is a cover off of C types.

      		
      
        

      		
    

    

      
        

          #
        

        
 Notice how all ooc types are capitalized? In fact, we cannot create variables with C types.

      		
      
        
chr = 'c' : char
// [ERROR] Undefined type 'char'

chr = 'c' : Char
// No error

      		
    

    

      
        

          #
        

        
 Covering is as simple as creating a new class. Notice that we can define functions for covers.

      		
      
        
String: cover from Char* {
  roll?: func -> Bool { this == "rock" }
}

if("rock" roll?()) "Rock n' Roll" println()
// Rock n' Roll

          #
        

        
 Once we have a cover already defined, there’s no need to specify what we are are covering from (the Int cover
is defined in the standard).

      		
      
        
Int: cover {
  seconds2minutes: func -> This { this / 60 }
}

2520 seconds2minutes() toString() println()
// 42

This n' this

      		
      
        

      		
    

    

      
        

          #
        

        
 Let us sidestep a bit to better understand the usage of This and this.
This is a reference to a type, in the seconds2minutes example above, This
refers to the Int type. Hence, we can say seconds2minutes is a function that
returns a value of type Int

      		
      
        

      		
    

    

      
        

          #
        

        
 this is the reference to the actual object of type This. In the example
above, this == 2520 and This == Int

      		
      
        

      		
    

    

      
        

          #
        

        
 When used in a class, This refers to the class while this refers to the instance
of that class

      		
      
        
Animal: class {
  animal: func -> This { return this } // Simply returns the object animal was called on (object is of type Animal)
}

Extending classes without subclassing

      		
      
        

      		
    

    

      
        

          #
        

        
 In ooc, it is possible to define additional behavior for a class without subclassing it, rather
by using covers and clever polymorphism.

      		
      
        
Human: class {
  height: Int
  init: func (=height)
}

me := Human new(6)

ExtendedHuman: cover from Human {
  printHeight: func {
    this as Human height toString() println()
  }
}

me as ExtendedHuman printHeight()
// 6
